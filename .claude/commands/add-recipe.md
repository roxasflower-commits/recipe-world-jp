# レシピ追加ルール

レシピをサイトに追加する際は、以下の3つが**全て完了してから**コミット・反映すること。

## 必須チェック項目

### 1. 掲載元URLの確認
- `curl` または WebFetch でHTTPステータスが **200** であることを確認する
- 403（bot block）の場合はブラウザで実際にページが開けることを別途確認する
- 404・その他エラーの場合は別のURLを探す
- **URLが確認できるまで recipes.ts に追加しない**

### 2. 掲載元のレシピ・材料を和訳
- 確認済みURLのページ内容を取得し、材料・手順を日本語に翻訳する
- `sourceUrl` に設定したURLの内容と `ingredients` / `instructions` が一致していること
- WebFetch で取得できない場合はChromeブラウザで内容を確認してから翻訳する

### 3. 写真の確認
- Unsplashを使う場合：`curl` でHTTP 200を確認してから設定する
- ローカル画像を使う場合：`public/images/` にファイルが存在することを確認してから設定する

## 適用シーン

- 単発でレシピを追加するとき
- 新しい料理ジャンルを作ってレシピを追加するとき
- トップシェフを増やしてレシピを追加するとき

## チェックリスト（追加前に確認）

- [ ] `sourceUrl` がHTTP 200で応答する（またはブラウザで確認済み）
- [ ] `ingredients` と `instructions` が `sourceUrl` 先のレシピを和訳したもの
- [ ] `image` のURLまたはローカルファイルが存在・表示できる

3つ全て ✅ になってから `recipes.ts` に追加し、TypeScript確認 → コミット → プッシュする。
